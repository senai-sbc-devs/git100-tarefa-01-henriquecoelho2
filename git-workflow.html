O ciclo de Fluxo de Trabalho do Git e GitHub

Esta é uma página web que explica o fluxo de trabalho do Git e GitHub para que você entenda o passo a passo de como trabalhar utulizando essas ferramentas

  • Clone

    • Inicie clonando o repositório remoto no seu repositório local ou criando um repositório local.

  • Branch

    • Crie uma nova branch para suas atualizações para que depois possa unificar todas as mudanças futuras na main.

  • Commit

    • Após feitas todas as modificações use o comando "git add ." e em seguida escreva um commit documentando todas as modificações feitas no código e de push para enviar todas as modificações feitas na sua branch para o repositório remoto

  • Pull-Request

    • Vá para o GitHub, acesse o repositório remoto e após verificar se todas as modificações feitas estão corretas, e em seguida crie um pull-request para que sua branch seja armazenada no repositório remoto junto da main

  • Merge

    • Após a revisão do pull-request anterior e o envio de comentários sobre as alterações feitas, o próximo passo é fazer o merge para que a main esteja atualizada com todas as mudanças mais recentes feitas.
